## 1. System Architecture (How it Works)

The platform is built on a modern **Full-Stack Integrated Architecture**:

1.  **Frontend (React + Tailwind V4)**: Captures user data through a 5-phase "Zero-Scroll" wizard. It manages the visual state and theme toggling.
2.  **Persistence (Supabase)**: When a user clicks "Finalize", the data is immediately secured in a PostgreSQL database for historical records and multi-session access.
3.  **Intelligence Engine (Python FastAPI)**: The "Brain" of the system. It receives the wizard data, performs complex tax mathematics in Python, and returns the optimized strategy to the dashboard.
4.  **Optimization Loop**: The frontend displays the backend's results (Institutional vs. Advanced) and highlights the "Net Retention" (money saved).

## 2. Input Parameters: Why We Collect Them

Every field in the wizard has a specific purpose in the tax formula:

### Phase 1: Identity & Profile
*   **Age Group**: Tax slabs change for Senior Citizens (60+) and Super Seniors (80+). 
*   **Employment Type**: Affects eligibility for certain deductions like Standard Deduction.
*   **City Type (Metro/Non-Metro)**: Crucial for **HRA Exemption**. Metros allow 50% of basic salary as exemption, while Non-Metros allow 40%.

### Phase 2: Salary Components (Income Source A)
*   **Basic Salary**: The primary base for all tax calculations and HRA logic.
*   **HRA (House Rent Allowance)**: Part of the salary intended for rent. It can be exempted from tax if you pay rent.
*   **Allowances & Bonus**: Fully taxable components that increase your Gross Total Income.

### Phase 3: Other Income (Income Source B)
*   **Interest Income**: Taxable under "Income from Other Sources."
*   **Rental Income**: Income from owning property (Taxed after a standard 30% repair deduction).
*   **Freelance Income**: Professional income (Business/Profession).

### Phase 4: Deductions (The Tax Shields)
*   **80C (₹1.5L Limit)**: Investments in PPF, ELSS, Insurance, etc. (Only for Old Regime).
*   **80D**: Medical Insurance premiums.
*   **80E**: Interest on Education Loans.
*   **NPS (₹50k Extra)**: National Pension Scheme additional deduction.
*   **Rent Paid**: Used along with Basic and HRA to calculate the **HRA Exemption** amount.

## 3. The Mathematics of Tax Calculation (FY 2024-25)

The system calculates tax using the **Step-Slab Method**.

### Institutional Regime (Old)
*   **Standard Deduction**: ₹50,000 is subtracted from Gross Income.
*   **Deduction Sum**: All Phase 4 inputs (80C, 80D, etc.) are subtracted.
*   **Tax Slabs**:
    *   0 - 2.5L: Nil
    *   2.5L - 5L: 5%
    *   5L - 10L: 20%
    *   10L+: 30%
*   **Rebate**: If taxable income ≤ 5L, Tax = 0.

### Advanced Regime (New - Budget 2024)
*   **Standard Deduction**: ₹75,000 is subtracted.
*   **Deductions**: None allowed.
*   **Tax Slabs**:
    *   0 - 3L: Nil
    *   3L - 6L: 5%
    *   6L - 9L: 10%
    *   9L - 12L: 15%
    *   12L - 15L: 20%
    *   15L+: 30%
*   **Rebate**: If Gross Income ≤ 7L, Tax = 0.

## 4. The Optimization (Delta Logic)

The final "Net Retention" is calculated as:
`Net Retention = |Old_Tax - New_Tax|`

The system identifies which path is lower and marks it with the **"+ OPTIMIZED"** badge on the Executive Synthesis report.

## 5. Security & Persistence
All inputs are saved to the `tax_profiles` table in Supabase. This ensures that even if the user refreshes or switches devices, their financial synthesis is preserved.
